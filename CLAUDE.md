# CLAUDE.md — noise-flow

This file provides guidance to Claude Code when working in this repository.

## What this is

`@elysian256/noise-wave-flow` is a standalone npm package that provides an organic
noise-field canvas animation with a wind-cursor effect. It was extracted from
the [Multioriontech website](https://multioriontech.com).

- **npm:** `@elysian256/noise-wave-flow`
- **GitHub:** https://github.com/rohit-sagar256/noise-wave-flow
- **Author:** Rohit Sagar — https://github.com/rohit-sagar256
- **Organization:** Multioriontech — https://multioriontech.com

---

## Architecture

```
src/
  core.js     Pure animation engine — no framework deps. Exports:
                createNoiseFlow(canvas, container, config) → { update, destroy }
                mountNoiseFlow(container, config)          → { canvas, update, destroy }
  index.js    Re-exports createNoiseFlow + mountNoiseFlow
  react.js    React hook. Exports:
                useNoiseFlow(config) → { canvasRef, containerRef, update }
dist/         Built output — generated by `npm run build`, do not edit manually
  index.js    ES module
  index.cjs   CommonJS
  react.js    React hook ES module
  react.cjs   React hook CommonJS
```

npm install       # install dev deps

## Publishing

```bash
npm run build
npm publish --access public
```

Bump `version` in `package.json` before publishing a new release. Use semver:
- Patch (`0.1.x`) — bug fixes, no API changes
- Minor (`0.x.0`) — new options or features, backwards compatible
- Major (`x.0.0`) — breaking API changes

## Key design decisions

### Single noise formula
All canvases — no matter how many are on a page — share the same 4-octave sine
noise formula. This is intentional: it makes multiple canvases look like one
continuous field when `worldOffset` is used.

### worldOffset: 'screen'
When set, `resize()` reads `canvas.getBoundingClientRect()` and uses those
viewport coordinates as the spatial offset in the noise formula. This makes a
sub-canvas (e.g. a sidebar panel) render the same slice of the field as if it
were part of the larger hero canvas.

### config is live-patchable
`update(patch)` does `Object.assign(cfg, patch)` — no restart. Keep config keys
stable; don't remove keys that the render loop reads every frame.

### React hook mounts once
`useNoiseFlow` runs `createNoiseFlow` on mount and `destroy` on unmount. Config
is snapshotted via `configRef` so passing an inline object won't cause
re-mounts. Use the returned `update()` for live changes.

## What NOT to do

- Do not add framework-specific logic to `core.js` — keep it pure DOM/Canvas API.
- Do not bundle `react` — it is a peer dependency and must stay external.
- Do not skip `npm run build` before publishing — `dist/` must be current.
- Do not commit `dist/` to git — it is in `.gitignore` and rebuilt on publish.
